問題 05：インターフェイスを使用することができる。
問 01
インターフェイスのメソッドに自動的に宣言されたことになっている修飾子をア〜オの中から答えなさい。
  • アpublic abstract
  • イprivate abstract
  • ウpublic final
  • エprivate final
  • オprotected abstract
💡 解説
✅ 正解：ア（public abstract）
インターフェイスで宣言されたメソッドは、記述がなくても自動的に public abstract として扱われます。つまり外部から呼び出し可能で、かつ実装を持たない抽象メソッドです。インターフェイスに private や protected のメソッドは原則として宣言できません。
問 02
インターフェイスに関する記述について、最適な記述をア〜オの中から答えなさい。
  • ア1つのクラスに1つだけインターフェイスをインポートすることができる。
  • イ1つのクラスに1つだけインターフェイスを実装することができる。
  • ウ1つのクラスにインターフェイスを実装する場合も単一継承でのみ実装することができる。
  • エ1つのクラスに複数のインターフェイスをインポートすることができる。
  • オ1つのクラスに複数のインターフェイスを実装することができる。
💡 解説
✅ 正解：オ
Javaではクラスの継承（extends）は単一継承のみですが、インターフェイスの実装（implements）は複数指定できます。例：class Hero implements Fightable, Flyable。「インポート」ではなく「実装（implements）」という点に注意しましょう。
問 03
インターフェイスのアクセス修飾子をア〜オの中から答えなさい。
  • アpublic と private のみ
  • イpublic と package private のみ
  • ウprivate と package private のみ
  • エpublic と protected のみ
  • オprotected と package private のみ
💡 解説
✅ 正解：イ（public と package private のみ）
インターフェイス自体のアクセス修飾子は public（公開）または修飾子なし（package private、同じパッケージのみ）の2種類のみです。private や protected はインターフェイスのトップレベル宣言には使用できません。
問 04
Hero クラスに Runnable インターフェイスを実装する時の、クラス宣言部分の最適な記述をア〜オの中から答えなさい。
  • アpublic class Hero extends Runnable
  • イpublic class Hero implements Runnable
  • ウpublic class Runnable extends Hero
  • エpublic class Runnable implements Hero
  • オpublic class Hero, Runnable
💡 解説
✅ 正解：イ
インターフェイスを実装する場合は implements キーワードを使います。extends はクラスの継承に使うキーワードです。また、クラス名は Hero、実装するインターフェイス名は Runnable なので、「Hero implements Runnable」が正しい記述です。
問 05
下記の説明を読み、空欄①に当てはまるクラス宣言を記述しなさい。

Hero クラスは Character クラスを継承し、Fightable インターフェイスと Flyable インターフェイスを実装する。
public class Hero ① { public void flying() { } public void fight() { } }
💡 解説
✅ 正解：extends Character implements Fightable, Flyable
クラスの継承は extends、インターフェイスの実装は implements を使います。複数のインターフェイスを実装する場合はカンマ区切りで並べます。
完全な宣言：public class Hero extends Character implements Fightable, Flyable
問題 06：オブジェクト概念（多態性など）を理解し、プログラミングを行うことができる。
問 06
Is-a の関係に従い、正しい関係には「○」を、誤った関係には「×」を記述しなさい。
関係 ○ / ×
ア．電話クラスを継承し、携帯電話クラスを宣言する。
イ．東京駅クラスを継承し、大阪駅クラスを宣言する。
ウ．社員クラスを継承し、管理職クラスを宣言する。
エ．領収書クラスを継承し、書類クラスを宣言する。
オ．商品クラスを継承し、価格クラスを宣言する。
💡 解説
✅ 正解：ア○　イ×　ウ○　エ×　オ×
Is-a の関係とは「AはBの一種である」が成り立つかどうかです。
ア：「携帯電話は電話の一種」→ ○
イ：「大阪駅は東京駅の一種」とは言えない → ×（同じ種類であり、継承関係ではない）
ウ：「管理職は社員の一種」→ ○
エ：「書類は領収書の一種」とは言えない → ×（逆の関係）
オ：「価格は商品の一種」とは言えない → ×
問 07
ダウンキャストについて下記に対する、最適な記述をア〜オの中から答えなさい。

Hero クラスを継承し、SuperHero クラスを宣言する。なお、「Hero hero = new SuperHero() ;」で SuperHero オブジェクトを生成していることとする。
  • アHero sh = (Object) hero ;
  • イHero sh = (Hero) hero ;
  • ウSuperHero sh = (SuperHero) hero ;
  • エSuperHero sh = (Hero) hero ;
  • オHero sh = (SuperHero) hero ;
💡 解説
✅ 正解：ウ
ダウンキャストとは、親クラス型の変数に格納されたオブジェクトを、子クラス型に変換することです。
hero は Hero 型で宣言されていますが、実体は SuperHero オブジェクトです。これを SuperHero 型として扱うために (SuperHero) hero とキャストします。
ア：Object型にキャストしており、これはアップキャスト。
イ：変数型と結果型が同じ Hero 型で、ダウンキャストではない。
エ：(Hero) にキャストしているが、変数が SuperHero 型なので不一致。
オ：(SuperHero) にキャストしているが、代入先が Hero 型なので不一致。
問 08
下記のプログラムを実行した時の結果を、ア〜オの中から答えなさい。
public class Parent { } public class Child extends Parent { } public class Main { public static void main(String[] args) { Parent p1 = new Parent() ; Parent p2 = new Child() ; System.out.print(p1 instanceof Child) ; System.out.print(", ") ; System.out.print(p2 instanceof Child) ; } }
  • アfalse, false
  • イfalse, true
  • ウtrue, false
  • エtrue, true
  • オ何も表示されない
💡 解説
✅ 正解：イ（false, true）
instanceof 演算子は「オブジェクトが指定したクラス（またはそのサブクラス）のインスタンスかどうか」を判定します。
p1 は new Parent() なので Child のインスタンスではない → false
p2 は new Child() なので Child のインスタンスである → true
よって出力は false, true です。
問 09
次頁のプログラムを実行した時に、「マネージャー」と表示されるように空欄①に当てはまるコードをア〜オの中からすべて答えなさい（完答）。
public class Staff { public void show() { System.out.println("スタッフ") ; } }
public class Manager extends Staff { public void show() { System.out.println("マネージャー") ; } }
public class Main { public static void main(String[] args) { ① staff.show() ; } }
  • アStaff staff = new Staff() ;
  • イStaff staff = new Manager() ;
  • ウManager staff = new Staff() ;
  • エManager staff = new Manager() ;
  • オObject staff = new Manager() ;
💡 解説
✅ 正解：イ・エ
「マネージャー」を表示するには show() が Manager クラスのメソッドとして呼ばれる必要があります。
ア：Staff のインスタンスなので「スタッフ」が表示される → ×
イ：Staff 型だが実体は Manager → ポリモーフィズムにより「マネージャー」 → ○
ウ：new Staff() を Manager 型に代入できない → コンパイルエラー → ×
エ：Manager のインスタンスなので「マネージャー」 → ○
オ：Object 型には show() メソッドがないためコンパイルエラー → ×
問 10
下記のプログラムの空欄②〜⑤に当てはまるコードを記述しなさい。
コンソールへの出力結果：にゃあ〜 / ワンワン
public abstract class Animal { public String name ; public int age ; public void showProfile() { System.out.println("名前は、" + name + "、" + age + "歳です。") ; } public ② void cry() ; }
public class Cat extends Animal { public void cry() { ③ ("にゃあ〜") ; } }
public class Dog extends Animal { public void cry() { ③ ("ワンワン") ; } }
public class Main { public static void main(String[] args) { Cat cat = ④ Cat() ; cat.name = "タマ" ; cat.age = 3 ; Dog dog = ④ Dog() ; dog.name = "ポチ" ; dog.age = 5 ; Animal[] animals = new Animal[ 2 ] ; animals[0] = cat ; animals[1] = dog ; for (Animal animal : animals) { ⑤ ; } } }
💡 解説
✅ 正解：②abstract　③System.out.println　④new　⑤animal.cry()
② abstract：抽象メソッドの宣言に必要。
③ System.out.println：文字列を出力するメソッド。
④ new：インスタンス生成キーワード。
⑤ animal.cry()：ポリモーフィズムにより、animal が Cat なら「にゃあ〜」、Dog なら「ワンワン」が表示されます。
問題 07：コレクションクラスを使用することができる。
問 11
コレクションに関する記述について、最適な記述をア〜オの中から答えなさい。
  • アList は格納した順番があり、格納した順番で値を取り出せる。
  • イList は格納した順番は当てにならず、値を取り出してみないとわからない。
  • ウSet は同じ値を重複して格納することができる。
  • エSet は必ず格納した順番で取り出せる。
  • オMap はキーと値のペアで格納し、値で検索できる。
💡 解説
✅ 正解：ア
コレクションの特徴：
List：順番あり・重複あり。格納した順番で取り出せる。
Set：順番なし（保証なし）・重複なし。
Map：キーと値のペアで格納。キーで検索する（値ではない）。
イ：List は順番が保証されているので誤り。ウ：Set は重複不可。エ：Set は順番保証なし。オ：Map は「キー」で検索する。
問 12
下記のプログラムで Hero クラスのインスタンスをコレクションに格納する時、空欄①に当てはまるコードをア〜オの中から答えなさい。
public class Main { public static void main(String[] args) { List<Hero> heros = new ArrayList<>() ; heros.①(new Hero()) ; } }
  • アup
  • イset
  • ウget
  • エput
  • オadd
💡 解説
✅ 正解：オ（add）
List（ArrayList）に要素を追加するメソッドは add() です。
get()：要素の取得。set()：指定インデックスの要素を置き換える。put()：Map で使うメソッド。up：そのようなメソッドは存在しない。
問 13
コレクションのインスタンスを生成する際に、下記のような「<データ型>」の部分の呼び名を記述しなさい。

List<String> data = new ArrayList<String>() ;
💡 解説
✅ 正解：型パラメータ（ジェネリクス）
<String> のような <データ型> の部分を「型パラメータ」と呼びます。この仕組みを「ジェネリクス（Generics）」といい、コレクションに格納できる型を指定することでコンパイル時に型の安全性を確保できます。
問 14
ラッパークラスを1つ答えなさい。
💡 解説
✅ 正解例：Integer / Double / Boolean / Character / Long など
ラッパークラスとは、プリミティブ型（基本データ型）をオブジェクトとして扱うためのクラスです。コレクションにはオブジェクトしか格納できないため、ラッパークラスを使ってプリミティブ型をオブジェクト化します。
例：int → Integer、double → Double、boolean → Boolean
問 15
問 10 の main メソッドの Animal 型配列を ArrayList に変更する時、鳴き声表示の拡張 for 文の部分をどのように対応すべきか、答えなさい。
💡 解説
✅ 正解：変更不要（for (Animal animal : animals) { animal.cry() ; } のまま）
配列を ArrayList<Animal> に変更しても、拡張 for 文の記述は同じです。
変更前：Animal[] animals = new Animal[2] ;
変更後：List<Animal> animals = new ArrayList<>() ;
animals.add(cat) ; animals.add(dog) ; で要素を追加し、拡張 for 文は for (Animal animal : animals) のまま使用できます。
問題 08：例外処理ができる。
問 16
プログラムの不具合について、最適な記述をア〜オの中から答えなさい。
  • ア文法エラーは言語仕様と異なる記述に対して発生するが、コンパイルにより自動的に修正される。
  • イ論理エラーは処理内容に関りがあると発生するが、インタプリタにより自動的に修正される。
  • ウ実行時エラーは動作テストで発生が予防できるため、動作テストをしっかり行う必要がある。
  • エ文法エラーと論理エラーは開発者の過失であり、実行時エラーは想定外の事態により発生する。
  • オ文法エラーと実行時エラーは言語仕様に合っていないコードがあるために発生する。
💡 解説
✅ 正解：エ
文法エラー：言語仕様に合わない記述。コンパイル時に検出される（自動修正はされない）。
論理エラー：プログラムは動くが、処理の内容が誤っている。開発者が気づいて修正する必要がある。
実行時エラー：ファイルが見つからない・ネットワーク断絶など、実行中に想定外の事態で発生する。例外処理で対応する。
問 17
例外の種類に関する記述の組み合わせで、最適な記述をア〜オの中から答えなさい。
  • アException 系例外：回復の見込みがある、catch すべき例外。
    RuntimeException 系例外：回復の見込みは必須ではなく、catch しなくても例外。
  • イException 系例外：回復の見込みはあるが、catch すべきではない例外。
    RuntimeException 系例外：回復は必須ではなく、catch しなくてもよい例外。
  • ウException 系例外：回復の見込みがある、catch すべき例外。
    RuntimeException 系例外：回復は必須ではないが、catch すべき例外。
  • エException 系例外：回復の見込みはあるが、catch すべきではない例外。
    RuntimeException 系例外：回復が必須ではないが、catch すべき例外。
  • オException 系例外：回復の見込みがある、catch すべき例外。
    RuntimeException 系例外：回復は必須であるが、catch できない例外。
💡 解説
✅ 正解：ア
Exception 系（検査例外）：コンパイラが catch または throws を強制する。回復の見込みがある例外（例：IOException）。
RuntimeException 系（非検査例外）：catch しなくてもコンパイルエラーにならない。主にプログラムのバグに起因する（例：NullPointerException、ArrayIndexOutOfBoundsException）。
問 18
下記のプログラムの空欄①〜⑤に当てはまるコードを記述しなさい。
public class Main { public static void main(String[] args) { FileWriter fw = null ; ① { fw = new FileWriter("data.txt") ; fw.write("Hello topia!") ; } ② (IOException e) { System.out.println("ファイルを開くことができません") ; } ③ { ④ { if(fw != null) { fw.close() ; } } ⑤ (IOException e) { System.out.println("ファイルを閉じることができません") ; } } } }
💡 解説
✅ 正解：①try　②catch　③finally　④try　⑤catch
① try：例外が発生する可能性のある処理を囲む。
② catch：例外が発生した時の処理。
③ finally：例外の有無に関わらず必ず実行される処理。
④ try：finally ブロック内でも close() に例外が発生する可能性があるため再度 try。
⑤ catch：close() の例外をキャッチする。
問 19
下記のプログラムは、上記の問 18 のプログラムに対して、Java 仮想マシンによって自動的に close メソッドが呼び出されるプログラムである。上記の問 18 のプログラムを参照して、下記の空欄①に当てはまるコードを記述しなさい。
public class Main { public static void main(String[] args) { try ( ① ) { fw.write("Hello topia!") ; } catch (IOException e) { System.out.println("ファイルを開くことができません") ; } } }
💡 解説
✅ 正解：FileWriter fw = new FileWriter("data.txt")
これは「try-with-resources」構文です。try() の括弧内でリソースを宣言・初期化すると、try ブロックの終了時に JVM が自動的に close() を呼び出してくれます。finally ブロックで手動 close する必要がなくなります。
問 20
例外の対処方法について、最も適切な記述をア〜オの中から答えなさい。
  • アメソッド宣言に発生する可能性のある例外を throw で宣言すると try-catch 文が不要になり、メソッドの呼び出し元に例外処理を委ねることができる。また、Java 仮想マシンに throws を使用した処理文を記述することで例外インスタンスを渡すことができ、プログラム側から例外的状況を伝えることができる。
  • イメソッド宣言に発生する可能性のある例外を throws で宣言すると try-catch 文が不要になり、メソッドの呼び出し元に例外処理を委ねることができる。また、Java 仮想マシンに throw を使用した処理文を記述することで例外インスタンスを渡すことができ、プログラム側から例外的状況を伝えることができる。
  • ウAPI に多くの例外クラスが用意されているため、開発者自身で独自の例外クラスを宣言する余地はない。
  • エAPI に多くの例外クラスが用意されており、継承階層を構成している。しかし、主要な例外クラスには final が付けられているため、開発者自身が既存の例外クラスを継承して独自の例外クラスを宣言する機会はほとんどない。
  • オ大規模なプログラム開発では、さまざまな例外的状況を思い浮かべ、独自の例外クラスを宣言することで、実行時エラーに対して共通の対処が可能になる。
💡 解説
✅ 正解：イ
throws（複数形）：メソッド宣言に付けて「このメソッドは〇〇例外を投げる可能性がある」と呼び出し元に伝える。
throw（単数形）：実際に例外インスタンスを投げる（発生させる）キーワード。
アは throw と throws が逆になっているため誤り。ウは独自の例外クラスは宣言できるので誤り。エは主要な例外クラスが final で継承できないという記述が誤り。
